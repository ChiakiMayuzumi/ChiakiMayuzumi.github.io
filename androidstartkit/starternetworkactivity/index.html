Using StarterNetworkActivity to make easier HTTP request

android-starter-kit

android-starter-kit is a fast development kit for Android developer,and this topic describes how to use StarterNetworkActivity to make your new activity’s HTTP requests easier.

Overview

To make the HTTP requests, we could use some third-party libraries such as retrofit 2.0,which is known as a type-safe HTTP client for Android and Java.

android-starter-kit simply encapsulates retrofit 2.0 in it so if you are familiar with retrofit 2.0,it won’t be difficult for you to use StarterNetworkActivity.

How to use

In android-starter-kit demo App ,LoginActivity shows how to use StarterNetworkActivity,so let’s see the source code in :

  com.smartydroid.android.kit.demo.ui.activity.LoginActivity

Preparations

Preparations must be done before using StarterNetworkActivity,which contains:

  • a proper entity class
  • get RetrofitBuilder with baseUrl
  • get apiService and your own HTTP service interface

a proper entity class

get ApiService and your own HTTP service interface

Retrofit turns your HTTP API into a Java interface,so write your own HTTP service interface using the defined entity:

public interface AuthService {

  /**
   * Login Interface
   *
   * @param phone 
   * @param password 
   * @return Call
   */
  @FormUrlEncoded @POST("/auth/login") Call<User> login(
      @Field("phone") String phone,
      @Field("password") String password);

  @GET("/user/profile") Call<User> profile();
}

ApiService is defined to get your own HTTP service or other HTTP service with retrofit:

public class ApiService {

  // create your HTTP interface with retrofit
  public static AuthService createAuthService() {
    return retrofit().create(AuthService.class);
  }
  
  // create other HTTP interface with retrofit
  public static FeedService createFeedService() {
    return retrofit().create(FeedService.class);
  }

  // get retrofit
  private static Retrofit retrofit() {
    return RetrofitBuilder.get().retrofit();
  }
}

The following override callbacks should be paid attention to:

  // call before request
  @Override public void startRequest() {
    showHud(“Logining...");
  }

  // call after request success with your defined entity class
  @Override public void respondSuccess(User data) {
    AccountManager.store(data);
    Snackbar.make(getWindow().getDecorView(), “Login Success”, Snackbar.LENGTH_SHORT).show();
    Intent upIntent = NavUtils.getParentActivityIntent(this);
    NavUtils.navigateUpTo(this, upIntent);
  }
  
  // call when request end
  @Override public void endRequest() {
    dismissHud();
  }
